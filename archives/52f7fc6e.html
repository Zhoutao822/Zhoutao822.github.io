SpringBoot-DubboAdmin2.7安装与配置

发表于 2020-02-15 分类于 Tips 阅读次数： Valine：
本文字数： 6.1k 阅读时长 ≈ 6 分钟

参考：

Apache ZooKeeper
dubbo-admin

Dubbo，阿里旗下非常优秀开源RPC框架，现由Apache接管，RPC框架由生产者提供服务，消费者使用服务，中间由服务注册中心管理，一般较为常见的开源服务注册中心为ZooKeeper，ZooKeeper中注册的服务没有可视化界面查看，需要通过命令行查看，而对于基于Dubbo的服务，可以使用dubbo-admin通过可视化界面进行控制。

1. 安装ZooKeeper

通过brew安装的ZooKeeper版本可能会比官网落后一些，通过zkServer start与zkServer stop启动与关闭ZooKeeper服务

2. 下载dubbo-admin并编译运行

dubbo-admin默认是处于develop分支，这个分支下默认的dubbo版本是2.7.3（最新的dubbo版本是2.7.5），且develop下的UI比之前版本更好看，但是功能不完善，而且对数据中心进行了划分。

下载完成后在项目的根目录下执行（maven项目，需要提前安装maven）

如果没有忽略测试部分，可能会报错

develop分支需要用到node，在编译UI部分的过程中可能会遇到

如果这个导致UI部分无法编译，可以执行npm install bcrypt --unsafe-perms。

如果成功编译，则会显示如下内容

由于我们下载下来并没有修改任何dubbo-admin的配置，因此会运行在8080端口，且ZooKeeper的地址和端口都为默认的127.0.0.1:2181。

我们可以在dubbo-admin-distribution/target下找到编译完成的jar包dubbo-admin-0.1.jar，然后直接运行即可

然后就可以在http://localhost:8080查看

3. 测试dubbo-admin

如果dubbo-admin正常运行，则我们编写的测试项目中生产者注册的服务以及消费者需要使用的服务会显示在网页中。
